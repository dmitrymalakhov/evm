# Исправление DNS кэша на Mac

## Проблема
DNS кэш на вашем Mac содержит неправильный IP адрес `198.18.0.16` вместо правильного `207.154.207.198`.

## Решение

### Вариант 1: Очистка DNS кэша через терминал

Откройте терминал и выполните:

```bash
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

Введите пароль администратора когда потребуется.

### Вариант 2: Перезагрузка сети

1. Откройте **Системные настройки** → **Сеть**
2. Выберите ваше активное подключение (Wi-Fi или Ethernet)
3. Нажмите **"Дополнительно"** → вкладка **"DNS"**
4. Удалите все DNS серверы и добавьте: `8.8.8.8` и `8.8.4.4` (Google DNS)
5. Нажмите **"OK"** и **"Применить"**

### Вариант 3: Проверка /etc/hosts

Проверьте, нет ли записи в файле `/etc/hosts`:

```bash
cat /etc/hosts | grep cyberelka
```

Если есть запись с `198.18.0.16`, удалите её:

```bash
sudo nano /etc/hosts
```

Удалите строку с `cyberelka2077.ru` и сохраните файл.

### Вариант 4: Временное решение

Пока DNS кэш не обновится, можно использовать IP адрес напрямую:

- Откройте в браузере: `http://207.154.207.198`
- Или добавьте в `/etc/hosts` правильную запись:
  ```bash
  sudo sh -c 'echo "207.154.207.198 cyberelka2077.ru" >> /etc/hosts'
  ```

### Проверка после исправления

После очистки кэша проверьте:

```bash
# Проверка DNS
nslookup cyberelka2077.ru 8.8.8.8

# Должен вернуть: 207.154.207.198

# Проверка доступности
curl -I http://cyberelka2077.ru
curl -I https://cyberelka2077.ru
```

## Почему это произошло?

IP адрес `198.18.0.16` - это адрес из диапазона `198.18.0.0/15`, который зарезервирован для тестирования сетей. Возможно:
- Ранее был настроен локальный тестовый сервер
- Использовался VPN или прокси
- Была запись в /etc/hosts

## Дополнительно

Если проблема сохраняется, попробуйте:
1. Перезагрузить Mac
2. Использовать другой DNS сервер (например, Cloudflare: `1.1.1.1`)
3. Проверить настройки VPN/прокси

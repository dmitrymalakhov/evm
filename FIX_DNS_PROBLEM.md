# Исправление проблемы с DNS

## Проблема
Ваш компьютер резолвит домен `cyberelka2077.ru` в неправильный IP `198.18.0.16` вместо правильного `207.154.207.198`.

## Причина
Скорее всего, VPN (Shadowrocket) или другой прокси перехватывает DNS запросы и подменяет ответы.

## Решение

### Шаг 1: Полностью отключите VPN
1. Откройте приложение Shadowrocket (или другой VPN)
2. Полностью отключите VPN (не просто переключите сервер, а именно выключите)
3. Убедитесь, что VPN не работает

### Шаг 2: Очистите DNS кэш на Mac
Выполните в терминале:

```bash
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

### Шаг 3: Проверьте файл /etc/hosts
Проверьте, нет ли там записи для `cyberelka2077.ru`:

```bash
cat /etc/hosts | grep cyberelka
```

Если есть запись с IP `198.18.0.16`, удалите её или закомментируйте (добавьте `#` в начале строки):

```bash
sudo nano /etc/hosts
```

### Шаг 4: Проверьте DNS настройки
1. Откройте **Системные настройки** → **Сеть**
2. Выберите активное подключение (Wi-Fi или Ethernet)
3. Нажмите **"Дополнительно..."** → вкладка **"DNS"**
4. Удалите все DNS серверы, которые выглядят подозрительно
5. Добавьте публичные DNS серверы:
   - `8.8.8.8` (Google DNS)
   - `8.8.4.4` (Google DNS)
   - или `1.1.1.1` (Cloudflare DNS)

### Шаг 5: Проверьте DNS резолвинг
После выполнения всех шагов проверьте:

```bash
nslookup cyberelka2077.ru 8.8.8.8
```

Должен вернуться IP `207.154.207.198`.

### Шаг 6: Перезапустите браузер
Полностью закройте и снова откройте браузер. Попробуйте открыть сайт в режиме инкогнито.

## Если проблема сохраняется

Если после всех шагов сайт все еще не открывается:

1. **Проверьте с другого устройства** (телефон через мобильный интернет, другой компьютер)
2. **Проверьте через онлайн-сервис**: https://www.isitdownrightnow.com/cyberelka2077.ru.html
3. **Временно добавьте запись в /etc/hosts**:
   ```bash
   sudo sh -c 'echo "207.154.207.198 cyberelka2077.ru" >> /etc/hosts'
   ```

## Важно
IP `198.18.0.16` - это зарезервированный IP адрес (RFC 2544), который часто используется VPN/прокси для перехвата DNS. Это подтверждает, что проблема в локальной настройке вашего компьютера, а не на сервере.

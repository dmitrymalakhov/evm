# Исправление перехвата DNS

## Проблема

Ваш компьютер резолвит домен `cyberelka2077.ru` в неправильный IP `198.18.0.15` вместо правильного `207.154.207.198`.

IP `198.18.0.15` - это зарезервированный адрес (RFC 2544), который используется VPN/прокси для перехвата DNS запросов.

## Решение

### Шаг 1: Полностью отключите VPN/Прокси

1. **Откройте приложение Shadowrocket** (или другой VPN/прокси)
2. **Полностью выключите VPN** (не просто переключите сервер)
3. **Закройте приложение** (не просто сверните, а закройте полностью)
4. **Проверьте, что VPN не работает:**
   ```bash
   ps aux | grep -i "vpn\|proxy\|shadowrocket" | grep -v grep
   ```
   Если есть процессы, завершите их:
   ```bash
   killall Shadowrocket
   ```

### Шаг 2: Очистите DNS кэш

```bash
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

### Шаг 3: Проверьте системные настройки DNS

1. Откройте **Системные настройки** → **Сеть**
2. Выберите активное подключение (Wi-Fi или Ethernet)
3. Нажмите **"Дополнительно..."** → вкладка **"DNS"**
4. **Удалите все DNS серверы**, которые выглядят подозрительно (особенно те, что начинаются с `198.18`)
5. **Добавьте публичные DNS серверы:**
   - `8.8.8.8` (Google DNS)
   - `8.8.4.4` (Google DNS)
   - или `1.1.1.1` (Cloudflare DNS)
   - или `1.0.0.1` (Cloudflare DNS)

### Шаг 4: Проверьте файл /etc/hosts

```bash
cat /etc/hosts | grep cyberelka
```

Если есть запись с IP `198.18.0.15` или `198.18.0.16`, удалите её:
```bash
sudo nano /etc/hosts
```

### Шаг 5: Перезапустите сетевые службы

```bash
sudo killall -HUP mDNSResponder
sudo dscacheutil -flushcache
```

### Шаг 6: Проверьте DNS резолвинг

```bash
dig cyberelka2077.ru @8.8.8.8 +short
```

Должен вернуться: `207.154.207.198`

```bash
dig cyberelka2077.ru +short
```

Тоже должен вернуться: `207.154.207.198`

### Шаг 7: Проверьте доступность сайта

```bash
curl -I http://cyberelka2077.ru
```

Должен вернуться: `HTTP/1.1 200 OK` или `HTTP/1.1 301 Moved Permanently`

## Если проблема сохраняется

Если после всех шагов домен все еще резолвится в `198.18.0.15`:

1. **Временно добавьте запись в /etc/hosts:**
   ```bash
   sudo sh -c 'echo "207.154.207.198 cyberelka2077.ru" >> /etc/hosts'
   ```

2. **Проверьте, нет ли других VPN/прокси приложений:**
   - ClashX
   - V2Ray
   - Surge
   - Proxyman
   - И другие

3. **Перезагрузите компьютер** - это поможет сбросить все сетевые настройки

## Важно

IP `198.18.0.0/15` (198.18.0.0 - 198.19.255.255) - это зарезервированный диапазон для тестирования межсетевого взаимодействия. Если ваш DNS резолвит домены в этот диапазон, это означает, что VPN/прокси перехватывает DNS запросы.
